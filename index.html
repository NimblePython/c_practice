Орбита спутника нестабильна, если скорость отличается от расчетной на более чем 0.1 м/с. Напишите программу, которая принимает расчетную (double) и фактическую (double) скорости, затем определяет, стабильна ли орбита.

№ 6
✎ СРЕДНЕ

Разделение экипажа

В космической экспедиции участвуют заранее подготовленные космонавты. Для предстоящего полета подготовлены модульные отсеки для космического корабля. Космическая программа профинансировала изготовление 3-х вариантов модулей, которые вмещают 2-х человек экипажа, 3-х или 4-х человек. По итогу отбора участников, будет выяволено общее число претендентов на экспедицию. Напишите программу, которая определит можно ли разделить число отобранных участников на равные группы, где каждая группа может содержать 2, 3 или 4 человека. На вход программа принимает количество космонавтов и выводит, можно ли их разделить без остатка, а также сколько потребуется полностью заполненных модулей и какого типа: 2-х, 3-х или 4-х местные. Программа должна рассчитать всевозможные варианты комплектаций модулями, исключая непоместившегося космонавта (если такой будет).

Определение секций для переменных

Изучи следующий код и определи, в каких секциях ELF-файла на диске будут находиться указанные переменные, функции и данные.

#include <stdio.h>
#include <stdlib.h>

// Глобальные переменные
int global_var = 42; 
int uninitialized_global_var; 

// Статическая переменная
static int static_var = 10;

// Структура
struct Point {
    int x;
    int y;
};

struct Point global_point = {1, 2};

// Функция
void print_message() {
    static int static_local_var = 5;
    int local_var = 3;
    printf("Message from function!\n");
}

int main() {
    int auto_var = 100;
    int *dynamic_var = malloc(4);
    if (dynamic_var) {
        *dynamic_var = 200;
    }

    struct Point stack_point = {3, 4};

    print_message();

    free(dynamic_var); 

    return 0;
}

Создание многофайлового проекта для космического корабля

Ты работаешь над программой управления космическим кораблем. Необходимо разделить функционал на несколько файлов:

  1. Файл navigation.c содержит функцию calculate_distance(float speed, float time) для расчета расстояния.
  2. Файл fuel.c содержит функцию check_fuel(int fuel_level) для проверки уровня топлива.
  3. Файл main.c — главный файл программы, который использует функции из navigation.c и fuel.c.
Требования:
  • Создай три файла: navigation.c, fuel.c и main.c.
  • Напиши заголовочный файл ship.h, в котором объяви прототипы функций.
  • В файле main.c запроси у пользователя скорость (float) и время (float), затем выведите рассчитанное расстояние.
  • Проверь уровень топлива (если меньше 10%, выведи предупреждение).
  • Скомпилируй проект с помощью GCC.
    • Дополнительное задание:
  • Используй команду nm для просмотра символов в объектных файлах до связывания и сравните их с символами в исполняемом файле после связывания.

🔹 Что нужно сделать?

Создай 3 файла:

  • main.c
  • file1.c
  • file2.c

🔹 Код файлов:

1️⃣ file1.c

#include 

int global_var = 10;  // Глобальная переменная, доступна из других файлов
static int static_var = 20;  // Видна только внутри file1.c

void print_from_file1() {
    printf("file1.c -> global_var = %d\n", global_var);
    printf("file1.c -> static_var = %d\n", static_var);
}
2️⃣ file2.c

#include 

extern int global_var;  // Объявляем глобальную переменную из file1.c
extern int static_var;  // Попробуем объявить static (должна быть ошибка!)

int global_var = 30;  // Попробуем переопределить (должен быть конфликт)

void print_from_file2() {
    printf("file2.c -> global_var = %d\n", global_var);
    printf("file2.c -> static_var = %d\n", static_var); // Должна быть ошибка компиляции
}
3️⃣ main.c

#include 

extern int global_var;
void print_from_file1();
void print_from_file2();

int main() {
    printf("main.c -> global_var = %d\n", global_var);
    
    print_from_file1();
    print_from_file2();

    return 0;
}

🔹 Шаги выполнения

  1. 1️⃣ Скомпилируй поэтапно: 
    gcc -c file1.c file2.c main.c
  2. 2️⃣ Исследуй таблицу символов через nm:

    nm file1.o
    nm file2.o
    nm main.o

    Что мы видим? Какие переменные есть? Видна ли static_var в file1.o?
    Есть ли global_var в двух .o?

  3. 3️⃣ Попробуй собрать программу: 
    gcc file1.o file2.o main.o -o program

    Получаешь ли **ошибку линковки** из-за global_var?

  4. 4️⃣ Исправь код:
    • Удали дублирующийся global_var = 30; в file2.c
    • Удали extern int static_var;
    • Перекомпилируй и снова проанализируй nm.
  5. 5️⃣ Посмотри, где хранятся переменные в памяти:

    objdump -t file1.o
    objdump -t file2.o
    objdump -t main.o

    В каком сегменте памяти: .bss, .data, .text?

  6. 6️⃣ Проверь строки в бинарнике: 
    strings program

    Видны ли строки из printf()?

🔹 Дополнительные шаги

После успешной компиляции выполни: 

objdump -x program

Проверь таблицу символов после статической компиляции: 

gcc -static file1.c file2.c main.c

🔹 Что смотреть?

  • Entry point — с какого адреса начинается выполнение программы.
  • Разделы памяти — .text, .data, .bss, .rodata.
  • Используемые библиотеки - (NEEDED) — какие .so загружаются динамически.
